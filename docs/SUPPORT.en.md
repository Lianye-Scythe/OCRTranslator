This document explains the recommended support, reporting, and contact paths for OCRTranslator.

## Read these first

Before opening an issue or sending email, please check these documents first:

- [README](README.en.md)
- [FAQ](FAQ.en.md)
- [Contributing](CONTRIBUTING.en.md)
- [Security Policy](SECURITY.en.md)
- [Changelog](CHANGELOG.en.md)

## General usage questions

If your question is about:

- installation / startup
- configuration
- supported provider / base URL styles
- selected-text capture, screenshots, overlays, or general usage

please review the [FAQ](FAQ.en.md) and [README](README.en.md) first.

## Bug reports

If you can reproduce a real application bug, please use the GitHub Issues bug template and include as much of the following as possible:

- reproduction steps
- runtime type (source / packaged exe)
- operating system and Python version
- relevant log / crash log excerpts
- screenshots or a short video for UI issues

## Feature requests

If you want to suggest:

- a new feature
- a UI / UX improvement
- API / provider support changes
- packaging / release workflow improvements

please use the GitHub Issues feature request template.

## Security issues

If the report involves:

- API keys
- private base URLs
- replayable requests
- sensitive logs
- vulnerability details that could be abused

please do not post it in a public issue. Contact: `po12017po@gmail.com`

Suggested subject: `[OCRTranslator Security] <short summary>`

## Discussions status

GitHub Discussions are currently disabled for this repository.

For now, the recommended public channels are:

- general questions / bugs / feature requests: GitHub Issues
- sensitive security reports: email

## Versions and downloads

Please treat the GitHub Releases page as the canonical public release channel:

- https://github.com/Lianye-Scythe/OCRTranslator/releases

The current public Windows package is still **unsigned**; formal code signing is planned for a future release.
